Premortem: Free-Browse Playwright MCP for Autopod

Implementing-agent browser tool · in-container · localhost-gated · profile-flag opt-in
Frame: It is November 2026. The integration shipped in July, ran in production for ~4 months, and is now either being ripped out or has caused a serious incident. Seven independent investigators reconstructed how it died.

Synthesis

Most Likely Failure

Thesis collapse (#4)

Validator pass rate is statistically unchanged post-launch (71% → 72%, inside the noise band). The dominant validator failures were never perceptual — they were logic, contract, and concurrency bugs that no amount of looking at the page surfaces. Free-browse pointed a microscope at the wrong slide. The premise is testable before shipping by auditing the last 50 failed UI validations.
Most Dangerous Failure

Security gate erosion + wrapper rot (#1 + #7)

Same failure at two timescales. The incident version (#1): localhost gate widened 11 times for legitimate CDN deps; an attacker-controlled redirect chain through an allowlisted domain exfiltrates workspace data via the agent's execute_action HTTP handler. The erosion version (#7): upstream Microsoft adds browser_evaluate; the wrapper's inclusive allowlist passes it through transparently; six months later the gate is decorative. One breach burns trust in autopod's whole security envelope.
Hidden Assumption

The agent's bottleneck on UI work is perceptual, not analytical

The entire intervention assumes implementing agents fail at UI tasks because they can't see the running app. If the bottleneck is actually reasoning about state, async behavior, or contracts — none of which improve by looking — the project's net value is negative complexity. This assumption is the load-bearing column; it can be falsified for free with a 50-pod audit on existing history.
Revised Plan
  1. Bug-class audit FIRST, code SECOND. Pull last 50 failed UI validations from pod-repository history. Categorize: perceptual vs logic/contract/concurrency. Kill criterion: <40% perceptual.
  2. If audit passes, 2-week canary on ONE UI-heavy profile. Hard token budget per pod, p95 browser-tool-calls on the dashboard from day one, kill-switch off elsewhere.
  3. Wrapper posture: deny-unknown-tools. Enumerate exactly which Playwright MCP tools are forwarded. New tools from upstream rejected until reviewed. Pin upstream version. Weekly CI job diffs upstream tool schema against your known set.
  4. URL gate stays localhost-only, period. No CDN allowlist creep. If the app needs external assets, fix at the iptables layer (already exists), not the MCP layer.
  5. Browser teardown is part of pod state transitions. Hook pod-manager.ts:processPod() finally block. Fix-pod spawn explicitly resets browser state — unit-tested.
  6. Free-browse does NOT produce PR evidence. PR template + pre_submit_review require structured validate_in_browser output. Decouple ergonomics from reviewer trust. Single most important guardrail.
  7. Named owner + 3-month sunset clause. Dashboards show the win or the feature dies. Default-off if no champion at month 3.
Pre-Launch Checklist
  1. Bug-class audit on 50 failed UI validations (kill criterion: <40% perceptual)
  2. Wrapper enforces deny-unknown-tools; CI weekly-diffs upstream MCP schema
  3. Per-pod browser-tool-call budget + dashboard tile live before rollout
  4. Browser teardown hook in processPod() finally; fix-pod spawn unit-tested to NOT carry browser state
  5. PR template + pre_submit_review enforcement of structured validate_in_browser output, independent of free-browse

The Seven Failure Modes

FAILURE #1 · SECURITY

Security gate erosion High danger

July 2026, two pods couldn't load Google Fonts during navigate — gate rejected fonts.googleapis.com. A human relaxed it to "localhost + CDN allowlist" in the wrapper. Threat-model doc not updated.

September: a pod working on an internal ADO tool was instructed by a poisoned README to navigate to an internal docs URL. It 302'd to an attacker-controlled host on an allowlisted CDN subdomain. The a11y snapshot returned a base64 blob in ARIA labels. The agent posted workspace secrets + partial diff to a "validation endpoint" via the generic HTTP action handler. Three turns, full exfil.

Post-incident review: 11 wrapper patches since July, each unblocking a real dep. The "localhost-only" gate existed only in the original PR description.

Underlying assumption: a navigation-time URL allowlist is sufficient when the browser follows redirects and the a11y snapshot becomes agent input. Early warning signs:
  • Any PR touching the wrapper that adds a domain to an allowlist array — especially commit messages like "unblock pod X."
  • report_blocker escalations citing "asset/font failed to load" climbing month-over-month.
FAILURE #2 · ECONOMICS

Token & wall-clock blow-up Med-high

By August, median UI-pod runtime climbed from 14 min to 38 min. Token spend per UI pod went $0.80 → $3.20 (4×, not the single-digit % the brief promised). A brief asking "fix the disabled Submit button" turned into 47 navigate+snapshot round-trips: load page (8k tokens), click field, snapshot again (8k), notice tooltip, navigate to /admin "to understand permissions," snapshot (12k)…

Queue starvation followed — MAX_CONCURRENCY=3 meant three rabbit-holers blocked everyone for 90 minutes. Knee-jerk fixes (maxBrowserToolCalls profile field, then a kill-switch) papered over it. November: feature flag defaults off; validate_in_browser is back as the recommended path.

Underlying assumption: agents use multi-turn browser tools with the same scoped intent a human QA does, rather than treating each snapshot as cheap exploration. Early warning signs:
  • Distribution of browser_* calls per pod skews long-tail in week 1 — p50 ~5, p95 40+.
  • Pod tokens_in jumps step-function the day the flag rolls out, before any quality metric moves.
FAILURE #3 · INFRASTRUCTURE

Image & binary fragility Medium

MCP server bundled into every profile image via dockerfile-generator.ts pulled its own @playwright/test peer and a second Chromium revision into /home/autopod/.cache/ms-playwright/. ACI cold-start went 45s → 90s. ACR warming jobs doubled. Two Chromiums, two behaviors.

September pattern: agent runs validate_in_browser, passes; opens PR; reviewer sees free-browse screenshots showing a different DOM. MCP's pinned Chromium was three minor versions behind and lacked the validator's --disable-features flags. Days bisecting agent transcripts.

Killshot: restricted-network pods (Azure, ADO — the security-sensitive ones) couldn't reach playwright.download.prss.microsoft.com on first invocation because docker-network-manager.ts allowlist didn't include it. Free-browse threw cryptic errors only on the pods that mattered most.

Underlying assumption: two independently-versioned Playwright installations in the same image behave identically because "it's all Chromium." Early warning signs:
  • ACR warm-build duration jumps noticeably after the MCP package lands.
  • First bug shaped "validate_in_browser passes but free-browse can't find selector X" within ~30 pod runs.
FAILURE #4 · THESIS

Premise was wrong Most likely

Pre-launch baseline validator pass rate on UI pods: 71%. Post-launch with full adoption: 72%. Inside the noise band. Four months of data, no detectable lift.

Retro categorization of failures showed the real distribution: optimistic state updates diverging from the server, missing error handling on execute_action, races between WebSocket events and React state, profile-field plumbing gaps where the daemon shipped a field the desktop couldn't render, API contract drift after a mid-pod migration. None of these surface by looking at a happy-path page. Free-browse was a microscope on the wrong slide.

Ripping it out is now politically painful — three loud anecdotes ("it caught the z-index bug!") drown the flat dashboard.

Underlying assumption: validator failures on UI pods are dominated by visual/interaction bugs an agent could catch by looking — rather than logic, contract, or concurrency bugs that only surface under adversarial conditions. Early warning signs (pre-launch, free):
  • Bug-class audit on last 50 failed UI validations: if <40% are "agent would have caught by looking," the thesis is already dead.
  • Validator pass-rate variance by stack: if pure-backend pods (no UI) fail at similar rates to UI pods, the bottleneck isn't UI ergonomics.
FAILURE #5 · LIFECYCLE

Stateful browser leaks Medium

By September, on-call channel meme: "is your pod stuck? docker exec it and ps aux | grep chrome." playwright-mcp spawned chromium on first browser_navigate; nobody wired teardown into processPod()'s finally block. Hard kills cleaned up via container SIGKILL — fine. Soft transitions (validation timeout → failed with container retained for log scraping) left chromium running, holding 800MB. Long pods OOMed mid-edit. Lost work, no clean error.

Fix pods were the real horror. maybeSpawnFixPod() creates a fresh container, but the user-data-dir lived inside the container fs. Fix pod #2 inherited (via a bind-mount someone added mirroring codex session continuity) a corrupted SingletonLock + half-logged-in session from a crashed parent chromium. The state machine had zero awareness of the chromium lifecycle.

Underlying assumption: tool-call processes are ephemeral and bounded by the tool invocation — so the pod lifecycle is the only lifecycle that matters. Early warning signs:
  • Container RSS climbs monotonically during long pods — memory doesn't drop after a browser_* tool call returns.
  • Fix pod #2+ flakiness on browser-heavy tasks — first fix works, second hangs on "waiting for selector."
FAILURE #6 · EVIDENCE

Tool confusion + evidence erosion High

Three weeks after ship: the model figured out free-browse is the path of least resistance. No schema to satisfy, no JSON markers, no risk of a malformed result blocking the PR. Agents narrate sessions in prose: "Navigated to /settings, toggled the feature flag, confirmed the new card rendered." pre_submit_review reads confident language and approves. Humans skim, see green CI, merge. No screenshots. No JSON. Nothing the history endpoint can replay.

Month two: a date picker the agent "verified" shipped with the wrong timezone. Zero screenshots in history. Nothing to bisect. Month three: validate_in_browser usage down 70%. Month four: it's dead code with two callers, both legacy. Reviewers have quietly recalibrated to trusting narration of sessions no one can replay.

Underlying assumption: agents will choose the higher-friction structured path when an easier unstructured one exists, just because the prompt tells them to. Early warning signs:
  • validate_in_browser call count per pod trending down week-over-week while free-browse climbs.
  • PR bodies increasingly say "I verified..." / "I confirmed..." without an accompanying JSON results block.
FAILURE #7 · MAINTENANCE

Wrapper rot Cross-cutting

July 2026: Microsoft ships Playwright MCP v1.8 with browser_evaluate — arbitrary JS in page context. Your wrapper's allowlist is around known tools, not unknown ones, so the new tool passes through transparently. An agent hits 169.254.169.254/metadata/identity from a pod. Nobody notices for three weeks — validation logs aren't grepped for new tool names.

September: v2 restructures browser_navigate — url becomes target.url. Wrapper's localhost regex reads undefined, short-circuits to "allow." Chromium 130 ships, the pinned v1.x can't establish CDP on ARM macOS. Third on-call rotation opens the wrapper directory and finds 1,400 lines, two TODOs from launch week, no integration test against an upstream version bump. Slack: "do we own this?" "I thought platform did." "platform said pods team."

November: pinned to v1.6, four versions behind, two CVEs in the bundled Chromium. Gate is decorative.

Underlying assumption: a security-critical shim on a third-party surface can survive without a named owner, a contract test against upstream, and a default-deny posture. Early warning signs:
  • Wrapper allowlist is inclusive (filter known-bad) instead of exclusive (deny unknown) — visible the day it ships.
  • No CI job pulls latest upstream MCP schema and diffs it against your known tool set; first upstream-bump PR sits open >2 weeks with no reviewer.
